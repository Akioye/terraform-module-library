Terraform Module Library

AWS Infrastructure Architecture · 3 Environments · 5 Modules · CI/CD Pipeline

5 Reusable Modules

  • • vpc — network, subnets, NAT, flow logs
  • • iam — roles, policies, IRSA/OIDC
  • • s3 — storage, encryption, lifecycle
  • • rds — Postgres, subnet group, SG
  • • eks — cluster, node groups, add-ons

3 Isolated Environments

  • • dev — t3.micro, SPOT, single NAT, no HA
  • • staging — t3.small, SPOT, mirrors prod config
  • • prod — t3.small, ON_DEMAND, deletion protection
  • • Separate VPC CIDRs: 10.0 / 10.1 / 10.2
  • • Separate state keys per environment

CI/CD Pipeline

  • • terraform plan on every PR
  • • Matrix: dev + staging + prod in parallel
  • • Posts results as PR comment
  • • All plans passed = required status check
  • • Remote state: S3 + DynamoDB locking
